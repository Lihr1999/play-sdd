# 用例生成约束规则

## 内容来源限制
- 仅依据用户提供的内容生成：需求文档、视觉稿、技术方案、需求评审会议记录、**OpenSpec 能力 spec**（`openspec/specs/<能力>/spec.md`）、**仓库内 Superpowers 实施计划**（`docs/superpowers/plans/*.md`），解析规则见 `input_sources.md`
- 不得引入上述范围之外的假设内容

## 埋点相关（严格剔除）
- **不生成**与埋点、数据统计、指标、监控相关的测试用例
- **不生成**验证埋点上报、埋点数据核对的用例
- 用例内容**不得包含**：埋点、曝光、点击UV、点击PV、上报埋点
- 注意事项**不得包含**埋点验证相关内容

## 反向/互斥场景（必须覆盖）

对每个有条件判断的功能点，正向和反向用例都必须生成：

| 正向场景 | 必须补充的反向场景 |
|---------|-----------------|
| 满足条件A → 触发行为X | 不满足条件A → 不触发行为X |
| 类型=A → 字段必选 | 类型=其他 → 字段可选/不必选 |
| 操作X → 触发流程Y | 操作非X → 不触发流程Y |
| 命中条件1（含条件2） → 通知A | 仅命中条件2 → 通知B |
| 数据存在 → 展示结果 | 数据不存在 → 提示/空态 |

**枚举数据分支**：当一个条件有多种数据形态时，逐一枚举每种形态：
```
示例：「近30天发文」的判断需拆分为：
  - 有图点评/图文/视频动态 → 算发文
  - 仅商单动态 → 算未发文
  - 仅无图点评 → 算未发文
  - 发布后已删除的动态 → 算发文
```

## 操作后状态验证（必须覆盖）

对每个写操作，除了验证操作本身，必须验证操作完成后的状态变化：

| 操作类型 | 必须验证的后续状态 |
|---------|-----------------|
| 新建/创建 | 列表中出现、状态字段正确、关联流程是否触发（如审批流） |
| 删除/下线 | 列表状态变更、操作按钮变化（如下线后变为「上线」按钮） |
| 审批通过 | 后台状态更新、C端状态同步 |
| 审批拒绝 | 后台状态更新、C端状态同步 |
| 审核中 | 相关操作按钮是否被禁用（不能编辑、删除、上下线） |

## 后台管理页面通用用例（必须补充）

需求涉及后台列表/管理页面时，除业务功能用例外，必须补充以下通用用例：

| 通用类型 | 必须覆盖的场景 |
|---------|-------------|
| 搜索/查询 | 单个查询、批量查询（注意分隔符）、查询项不存在时的提示 |
| 筛选 | 单个筛选条件、多条件组合筛选、默认值验证 |
| 分页 | 每页条数正确、翻页正常 |
| 批量操作 | 选中执行、上限边界值、超出上限提示 |
| 导出 | 有数据时字段正确、数据为空时的导出行为 |
| 操作后状态 | 操作成功后列表状态/按钮变化 |

## 文档分析时的重点标记

- `新增`/`修改`/`取消`/`删除` 标注的功能点 → 重点覆盖
- 涉及枚举值/类型（如达人类型、申请渠道）→ 每个枚举值单独出用例
- 涉及数量/比例（如50%、100个）→ 生成边界值用例
- "待确认"/"链接待补充"等标注 → 在对应用例中添加注释说明

## 覆盖度自查（生成后逐项确认）

- [ ] 每个正向场景是否有对应的反向/互斥场景？
- [ ] 有条件判断的数据分支是否逐一枚举？
- [ ] 每个写操作是否验证了操作后的状态变化？
- [ ] 后台列表页是否覆盖：单个查询、批量查询、查询项不存在、分页、组合筛选？
- [ ] 有数量上限的功能是否验证：边界值、超出上限的提示？
- [ ] 有状态流转的功能是否覆盖所有状态节点及操作权限变化？
- [ ] 导出功能是否区分：有数据、空数据两种场景？
- [ ] 有文案/链接变更的功能是否验证了新文案内容和跳转目标？

## 特定场景测试方法论

根据需求类型，选择对应规则文件（模式判断见 `rules/mode_selection.md`）：

- **服务端接口测试**（接口字段由多条件组合决定）：`rules/server_interface_rules.md`
- **A/B 实验功能测试**（实验参数控制 UI/交互）：`rules/ab_experiment_rules.md`
- **客户端功能测试**（默认）：本文件规则即适用
