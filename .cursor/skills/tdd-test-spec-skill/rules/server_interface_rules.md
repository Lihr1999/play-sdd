# 服务端接口测试方法论

当需求描述的是**接口返回值由多个输入条件组合决定**时（如推荐接口的人群分发逻辑），使用以下方法论。

## 识别信号

满足以下任一条件，切换为服务端接口测试模式：
- 需求描述某个接口字段（bool/枚举）由多个条件共同决定
- 存在"A且B则出/不出"的复合条件逻辑
- 存在异常兜底规则（字段缺失/非枚举值 → 视为某默认值）
- 测试方案明确说明通过 mock 服务构造算法输入

## 用例设计三步法

**第一步：抽取条件，建立判定表**
- 识别所有独立决策条件（C1, C2, C3...）
- 穷举所有有效组合，每个组合对应一条规则（R1, R2...）
- 标注每条规则的预期输出

**第二步：等价类划分**
- 对每个枚举型条件，每个枚举值单独一个等价类
- 异常值（字段缺失、非枚举值）视为独立等价类
- 边界值（字段为空字符串、null）视为独立等价类

**第三步：错误推测**
- 字段缺失 → 验证兜底逻辑
- 非枚举值 → 验证兜底逻辑
- 条件优先级 → 验证高优先级条件覆盖低优先级

## 用例表格格式（单一合并表格）

所有用例合并为一张横向表格，不拆分为多个竖表：

```markdown
| 用例ID | 测试项 | 设计方法 | Mock: 字段A | Mock: 字段B | 预期结果 | 优先级 |
|--------|--------|---------|------------|------------|---------|--------|
| TC-001 | ...    | 等价类   | `"值"`     | 任意值      | `true`  | P0     |
```

- Mock 的每个输入字段单独一列（字段名写在列头 `Mock: xxx`）
- 设计方法列填写：等价类 / 判定表 / 错误推测
- 预期结果列只写字段值，如 `true` / `false` / `"枚举值"`
- "任意值"表示该字段不影响本条规则的判断结果
