# 服务端接口测试示例：

> 此示例展示：接口返回字段由多条件组合决定时的测试设计方案

## 一、测试概述

### 1.1 测试范围
- 推荐流接口 `selectorTargetPopulation` 字段在各条件组合下的返回值正确性

### 1.2 接口说明
- **接口**：`/test/v1/recommend/all/feed`
- **被测字段**：`selectorTargetPopulation`（bool）—— `true` 出口味选择器，`false` 不出
- **依赖算法输入字段**：
  - `displayCrowd`（string，逗号分隔）：`1=新用户, 2=低心智, 3=中心智`
  - `qac_user_level_detail_1d`：`高活人群 / 中活人群 / 待激活人群C1 / 待激活人群C2`
- **异常兜底**：`qac_user_level_detail_1d` 未返回或异常时，兜底算「高活人群」

### 1.3 核心判断逻辑

```
IF displayCrowd 包含 "1"  → selectorTargetPopulation = true
ELSE IF displayCrowd 包含 "2" AND qac_user_level_detail_1d = "高活人群"  → true
ELSE  → false

异常兜底：qac 字段未返回/异常 → 等价于 qac = "高活人群" 处理
```

### 1.4 测试设计方法论说明

| 方法 | 应用场景 |
|------|---------|
| 判定表法 | 穷举 displayCrowd × qac 两个条件的所有有效决策路径 |
| 等价类划分 | displayCrowd（含1 / 仅含2 / 不含1不含2）；qac（高活 / 非高活枚举值 / 空/异常） |
| 错误推测法 | qac字段缺失、空值、非枚举值的兜底行为 |
